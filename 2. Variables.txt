			Отличия VAR, LET, CONST


 	Обьявление переменной называется декларация : "Declaration".			Пример:  var declaration ;

	Добавление значаения в переменную называется Инициализация "Initialisation"		Пример:  declaration = "Initialisation";

	Попытка получения значения переменной (в консоли) называется "Логирование"		

	JavaScript Блок - Блок это область между фигурными скобками. - if (...) {  // block }, function () {  //block   }



		Область видимости

	В JavaScript существует понятие Область Видимости. Область видимости определяет где в коде программы будут доступны переменные и функции.

	В JavaScript существует две Области Видимости - Локальная и Глобальная


	1) Локальная. Если переменнная создается(декларируется и инициализируется) внутри декларирования функции, то ее область видимости будет определяться как локальная и она будет доступна только в этой и вложенных в нее функциях.

	2) Глобальная. Если создается в непосредственно JS документе (вне функций и блоков). В данном случае она будет доступна везде в JS документе.



	*** ОТЛИЧИЯ МЕЖДУ VAR, LET и CONST:

		VAR - Область: функция,  пере-ДЕКЛАРИРУЕТСЯ, пере-ИНИЦИАЛИЗИРУЕТСЯ.

		LET - Область: БЛОК, НЕ пере-ДЕКЛАРИРУЕТСЯ, пере-ИНИЦИАЛИЗИРУЕТСЯ.

		CONST - Область: БЛОК, НЕ пере-ДЕКЛАРИРУЕТСЯ, НЕ пере-ИНИЦИАЛИЗИРУЕТСЯ.







	Переменные в функциях(локальные) живут во время выполнения функции(доступны только в фуцнкции). После заканчивания работы функции, они 'вымирают'.






	PS - В других языках программирования в основном область всех переменных является блок, это спецефичность JS-a. 

		Поэтому рекомендуется использовать ключевые слова let и  const. Чтобы привыкнуть к их поведению.


	PS - когда мы присваиваем массив к const, мы можем применить метод push() и добавлять туда данные, не смотря на то что это константа, но как мы знаем мы не можем присваивать к нему другое значение

	Также, если мы присваиваем обьект к const, мы не можем присваивать к нему другое значение, но можем менить и присваивать значения - к значениям обьекта.	e.g. object.keyWord = value


	PS - Лучше использовать const - там где это возможно, чтобы увеличить скорость нашего кода и оптимизацию.
